Semejanzas y diferencias entre XML y JSON

Aspecto Semejanzas Diferencias
Formato Ambos son formatos de texto legibles para humanos y máquinas. XML utiliza etiquetas de apertura y cierre, mientras que JSON utiliza llaves y corchetes.
Soporte Ambos son ampliamente utilizados en el intercambio de datos. XML es más utilizado en entornos empresariales y lenguajes como SOAP, mientras que JSON es comúnmente usado en aplicaciones web y RESTful APIs.
Tipos de datos Ambos pueden representar datos simples y complejos. XML permite la definición de tipos de datos complejos mediante esquemas, mientras que JSON se basa en estructuras simples como cadenas, números, booleanos, arrays y objetos anidados.
Legibilidad Ambos son legibles para los desarrolladores debido a su estructura jerárquica. JSON es generalmente más legible y conciso que XML.
Extensibilidad Tanto XML como JSON permiten la extensión mediante el uso de etiquetas o propiedades personalizadas. XML tiene un mecanismo más formal para la extensibilidad a través de la definición de esquemas y namespaces, mientras que en JSON la extensibilidad es más flexible pero menos estructurada.
Compatibilidad con APIs Ambos formatos son compatibles con una amplia variedad de lenguajes de programación y tienen librerías y herramientas disponibles para su manipulación. Algunos lenguajes y frameworks pueden tener mejor soporte nativo para XML o JSON, lo que puede influir en la elección del formato a utilizar.
Facilidad de uso JSON es generalmente más fácil de usar y comprender para los desarrolladores debido a su sintaxis más concisa. XML puede resultar más verboso y complicado de escribir y analizar en comparación con JSON.

XML


XML (Extensible Markup Language) es un lenguaje de marcado utilizado en sistemas web para estructurar, almacenar y transportar datos. Permite representar información legible tanto para humanos como para máquinas. XML es flexible y extensible, lo que permite definir estructuras de datos personalizadas. Se utiliza para intercambiar datos entre aplicaciones, configurar servicios web, almacenar información estructurada en bases de datos y más. XML es independiente del lenguaje y la plataforma, lo que facilita el intercambio de datos entre diferentes sistemas y aplicaciones web.

    JSON

    JSON (JavaScript Object Notation) es un formato de intercambio de datos utilizado en sistemas web. Es ligero, legible para humanos y fácilmente interpretable por las máquinas. JSON se basa en una estructura de pares clave-valor y se utiliza ampliamente para transmitir datos entre aplicaciones web. Es especialmente popular en el desarrollo de servicios web RESTful y APIs. JSON permite representar de manera eficiente datos estructurados como objetos, arreglos, cadenas de texto, números y valores booleanos. Además, es compatible con una amplia variedad de lenguajes de programación, lo que lo hace muy versátil en entornos web.

    Codigo JS

    
      // Cargar el archivo JSON
      fetch("peliculas.json")
        .then(response => response.json())
        .then(data => mostrarPeliculas(data));
    };
  
    function mostrarPeliculas(data) {
      var listaPeliculas = document.getElementById("lista-peliculas");
  
      data.peliculas.forEach((pelicula) => {
        var titulo = pelicula.titulo;
        var director = pelicula.director;
        var anio = pelicula.anio;
        var genero = pelicula.genero;
        var li = document.createElement("li");
        var texto = document.createTextNode(titulo + " - " + director + " (" + anio + ") - " + genero);
        li.appendChild(texto);
        listaPeliculas.appendChild(li);
      });
    }

      AJAX

      AJAX (Asynchronous JavaScript and XML) es una técnica utilizada en sistemas web para realizar comunicaciones asíncronas entre el cliente y el servidor. Permite cargar y actualizar datos en una página web sin tener que recargarla por completo. AJAX combina varias tecnologías, como JavaScript, XML o JSON y el objeto XMLHttpRequest, para enviar y recibir datos en segundo plano. Esta técnica mejora la experiencia del usuario al proporcionar respuestas rápidas y dinámicas. Permite cargar contenido adicional, enviar formularios y actualizar partes específicas de una página sin interrumpir la interacción del usuario.

      Codigo AJAX

      
        // Cargar usuarios al cargar la página
        window.onload = function() {
          cargarUsuarios();
        };
    
        function cargarUsuarios() {
          var xhttp = new XMLHttpRequest();
          xhttp.onreadystatechange = function() {
            if (this.readyState === 4 && this.status === 200) {
              var data = this.responseXML;
              mostrarUsuarios(data);
            }
          };
          xhttp.open("GET", "usuarios.xml", true);
          xhttp.send();
        }
    
        function mostrarUsuarios(data) {
          var listaUsuarios = document.getElementById("lista-usuarios");
          var usuarios = data.getElementsByTagName("usuario");
    
          for (var i = 0; i < usuarios.length; i++) {
            var nombre = usuarios[i].getElementsByTagName("nombre")[0].textContent;
            var email = usuarios[i].getElementsByTagName("email")[0].textContent;
    
            var li = document.createElement("li");
            var texto = document.createTextNode(nombre + " - " + email);
            li.appendChild(texto);
            listaUsuarios.appendChild(li);
          }
        }
